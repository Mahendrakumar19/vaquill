Title: Enable optional Markdown rendering (fix '**' bold not working)

Summary
-------
Users expect Markdown bold (`**bold**`) to render in case/argument/judgment text, but the frontend currently displays plain text. No markdown renderer (e.g., `react-markdown`) is installed or used. This issue proposes adding an optional "Enable Markdown" toggle at the start (case creation / argument input) and rendering displayed text as Markdown when enabled.

Problem
-------
- Searching the frontend shows no `react-markdown` / `remark-gfm` usage.
- Text is displayed as plain text; `**` is not converted to bold HTML.

Goal
----
- Add an optional toggle so users (or the judge) can enable Markdown rendering for case descriptions, arguments, and judge responses.
- Default remains plain text for safety; toggling enables GitHub-flavored Markdown (GFM) rendering.

   - `remark-gfm`

   ```powershell

   npm install react-markdown remark-gfm
   ```

2. Add a checkbox/toggle in UI:

   - Secondary location: `ArgumentPanel` input area for per-argument override (optional)
   - Persist the choice in local component state and include in case metadata if desired

3. Update rendering components to conditionally render Markdown:
   - Files to update:
     - `frontend/src/components/JudgmentDisplay.tsx` (judgment.reasoning)
     - `frontend/src/components/ArgumentPanel.tsx` (argument text and judge response)
     - `frontend/src/components/CaseView` or where case descriptions are shown (e.g., `App.tsx` / `CaseDetail`)

   - Example rendering snippet:
   ```tsx
   import ReactMarkdown from 'react-markdown';
   import remarkGfm from 'remark-gfm';

   // when markdownEnabled is true
   <ReactMarkdown remarkPlugins={[remarkGfm]}>{text}</ReactMarkdown>

   // when false (current behavior)
   <div>{text}</div>
   ```

   - Consider sanitization if you enable raw HTML (not recommended).
